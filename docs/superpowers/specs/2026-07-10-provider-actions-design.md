# Provider 操作收敛设计

## 目标

管理员打开 Providers 页面后，只看到当前状态下真实可执行的操作。已经保存的 IHG Session 必须复用，不能再次引导用户进入授权循环；暂未接入真实适配器的集团不得显示会产生误导的同步按钮。

## 页面行为

IHG 卡片根据 `connectionStatus` 渲染：

- `expired`、未配置或无账号：显示“开始 IHG 授权”和“查看日志”。
- 远程授权任务运行中：显示“继续授权”“停止授权”和“查看日志”。
- `session_authorized`：显示“测试连接”“同步90天价格”和“查看日志”。
- `active`：显示“测试连接”“重新同步90天价格”和“查看日志”，并展示最近同步时间、真实酒店数和价格数。
- `disabled`：仅显示“重新授权”和“查看日志”。

Marriott、Hilton、Hyatt、Accor 在没有可用真实同步适配器时只展示状态和“查看日志”，并明确显示“暂未接入真实同步”，不提供占位授权或同步操作。

## 删除范围

从 Providers 页面删除：

- 重复的 `providerCard` 实现。
- “登录账号”凭据保存入口。
- “人工授权”占位入口。
- 旧的“Playwright授权”入口。
- 手工粘贴并保存 Session 的入口。
- 在 Session 已保存状态下仍出现的 VNC 登录入口。

后端现有安全路由与加密 Session 数据保持不变，避免破坏已保存授权；本次只收敛可见操作和成功后的直接操作路径。

## 同步结果

点击“同步90天价格”后：

- 按钮进入禁用状态，避免重复提交。
- 成功时显示本次真实酒店数、真实价格数和完成状态，并刷新 Providers 数据。
- 失败时显示后端返回的真实失败原因；不得写入或展示 Mock 价格。
- 提供“查看同步任务”和“查看日志”入口。
